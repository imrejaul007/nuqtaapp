# NUQTA PRICING MODEL - FINAL VERSION
## Complete Merchant Package Structure

---

# CORE PRINCIPLES

## 1. No Cash Burn Policy
- All rewards are **merchant-funded**
- Nuqta never subsidizes coins
- Ticket cashback = auto-deducted from merchant commission

## 2. Payment Gateway Fees
- PG fees depend on payment gateway partner (varies)
- **Engine A:** PG fee INCLUDED in commission
- **Engine B:** PG fee ADDITIONAL to commission (transparent)

## 3. Prive Coin
- 100% merchant's decision
- No minimum, no maximum enforced
- Merchant sets their own rate

## 4. Coin Redemption
- **Minimum purchase: AED 300** to redeem any coins
- No cap on how much can be redeemed
- Below AED 300 purchase = coins cannot be used

# COIN SYSTEM RULES

## 1. Nuqta Coin (Universal)

| Rule | Details |
|------|---------|
| **Earn at** | Any merchant |
| **Spend at** | Any merchant in Nuqta network |
| **Minimum purchase to redeem** | AED 300 |
| **Expiry** | No expiry |
| **Funded by** | Merchant (from commission) |

---

| Rule | Details |
|------|---------|
| **Earn at** | Specific merchant |
| **Spend at** | ONLY at that merchant |

| **Funded by** | Merchant (optional add-on) |

## 3. Event Coin (Event-Wide)

| Rule | Details |
|------|---------|
| **Earn at** | Any merchant at event |

| **Funded by** | Merchant (from 5% mandatory commitment) |

| Rule | Details |
|------|---------|

| **Spend at** | Anywhere + exclusive perks |
| **Minimum purchase to redeem** | AED 300 |
| **Funded by** | Merchant (100% their choice) |

# REDEMPTION RULES

## Minimum Purchase Requirement

| Rule | Details |
|------|---------|
| **Threshold** | AED 300 |
| **Below AED 300** | Cannot redeem any coins |
| **At/Above AED 300** | Can redeem any amount of coins |
| **No cap** | User can redeem full coin balance if purchase qualifies |

### Example:
```
User has: 500 Nuqta Coins (AED 500 value)

Purchase AED 200:
→ Cannot redeem coins
→ Earns new coins normally

Purchase AED 300:
→ Can redeem up to AED 500 in coins
→ Pays: AED 0 (if using all coins) or partial
→ Still earns new coins on any cash portion paid
```

# TICKET CASHBACK MODEL (Events)

## How It Works (No Burn):

```
STEP 1: User buys ticket
├── Ticket price: AED 100
├── User receives: 100 Event Coins
└── Nuqta cost: AED 0 (coins are digital tokens)

STEP 2: User shops at event
├── Minimum purchase: AED 300 (to unlock redemption)
├── User wants to redeem: 100 Event Coins

STEP 3: Merchant pays
├── Purchase: AED 500
├── User uses: 100 coins (AED 100 value)
├── User pays cash: AED 400
├── Merchant commission: 7% of AED 500 = AED 35
│   ├── Event Coin: 5% = AED 25 (funds future coins)
│   └── Nuqta Revenue: 2% = AED 10

WHO FUNDED THE CASHBACK?
└── The merchant, through their mandatory 5% Event Coin commitment
└── Nuqta spent: AED 0
```
